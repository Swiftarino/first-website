Naming Conventions

Rules for naming files and paths

  • All lowercase letters
  • No spaces
  • Only letters, numbers, and dashes (-)

Examples: main.css | action-figure.jpg | my-first-website.html

Commit messages

When commiting to GitHub make sure to...

  • Start with a capital letter and the rest all lowercase
  • Do not end with a period
  • Action word first then what you performed that action on for the subject

Examples: Update code | Fix index.html | Add style.css

Structuring Content

Header

Use the element header

This element is used for the introductory content usually located at the top of the page.

Things that might be in this section include a logo, main navigation to other parts of page.

Footer

Use the element footer

This shows up at the end of the page

This usually contains a list of links, copyright information, extra info about the company or terms of services

Main

Use the element main

This element is for the main content on the page and only used once.

This tells the browser where the main content is

Section

Use the element section

This element is used to wrap related pieces of content together.

Each section has its own heading usually.

Article

Use the element article

This element specifies independent, self contained content.

An article should make sense on its own from the rest of the site. These include forum posts, blog posts or news stories.

Navigation

Use the element nav

This element represents a section of the page that navigates users with links to other parts on the page or direct users to other external documents

Landmark Roles

This is to help screen readers jump directly to specific sections on the page

Header

header role="banner" This defines the masthead, added to header

Nav

nav role="navigation" This defines the primary navigation, added to the top nav

Main

main role="main" This defines the main content of the page, added to main

Aside

aside role="complimentary" This defines an element related to the primary content, added to aside

Footer

footer role="contentinfo" This defines an element as info about the content (copyright, terms etc), added to footer

Aria-label

aria-label="" is used to provide the user with a non visual label that is announced by screen readers

Skip links

Skip links are specifically at the top of the page and when clicked can jump the user to different specified spots on the page

Font Properties

Font Family

This specifies the font for an element

Example: "Times New Roman", Times, serif;

Font Size

This adjusts the size of the font. Use a number attached to a unit of measurement (rem [best choice], em, px)

Example: 4rem, 1375em, 13px

Font Weight

This sets how thick or thin the font will be when displayed

Example: bold, normal, 900

Font Style

This sets the style for the text

Example: italic, normal

Line height

This adjusts the line height. A single number is used that multiplies the font size by that number

Example: 2.5

Font Shorthand

This adjusts all the different properties in a font at once

Example: font: italic 1.4em "Fira Sans", serif;
